# weight-logger-mini

初心者向けの体重管理アプリを、段階的に作るためのミニプロジェクトです。

## アプリの目的

- 日々の体重を入力し、保存して、履歴を見返せるようにする
- ChatGPT に貼り付けるための報告文を作れるようにする
- ただし、最初から作り込みすぎず、学習しやすい最小構成から進める

## Task 003 で作った範囲

- `index.html` / `styles.css` / `script.js` の最小構成を作成
- 画面の骨組みとして以下を配置
  - アプリタイトル
  - 今日の体重入力セクション（日付・体重・保存ボタン・報告文作成ボタン）
  - 記録一覧セクション
  - ChatGPT報告文セクション
- `script.js` に今後使う基本関数の枠を用意
  - `getTodayDate()`
  - `loadRecords()`
  - `saveRecord()`
  - `renderRecords()`
  - `buildChatGPTReport()`

## Task 005 で実装した範囲（最小実用化）

- 保存ボタンで日付・体重を `localStorage` に保存
- 保存直後に記録一覧を再描画
- ページ再読み込み後も保存データを復元
- 入力値が空、または体重が不正値のときは保存しない
- ChatGPT報告文を保存済みデータから自動生成
  - 初期表示時に反映
  - 保存後にも即時更新
  - 「報告文作成」ボタンでも再生成可能

## Task 006 で実装した範囲（記録管理の改善）

- 記録一覧を日付の新しい順で表示
- 各記録に「削除」ボタンを追加し、1件ずつ削除可能
- 削除直後に記録一覧と ChatGPT 報告文を即時更新
- 削除結果を `localStorage` に反映し、再読み込み後も維持
- 初期表示でサンプルデータを使わず、保存済み実データのみを表示
- 記録が0件のときに空状態メッセージを表示


## Task 007 で実装した範囲（入力体験の改善）

- 保存成功時にメッセージを表示（新規保存 / 同日上書き保存を明確化）
- 入力エラー時に原因別メッセージを表示（日付未入力・体重未入力・不正値）
- 同じ日付を保存する場合は、上書き確認ダイアログを表示
  - OK: 既存レコードを更新（追加せず置き換え）
  - キャンセル: 既存レコードを保持して保存中止
- 既存データに同日重複がある場合、初期読み込み時に日付ごとに1件へ正規化し、以降の保存でも同日重複が発生しないように修正
- 保存後はフォームを自然な状態に整理
  - 日付を今日に戻す
  - 体重入力を空に戻す
  - 体重入力欄にフォーカス
- 入力内容が不十分なときは保存ボタンを無効化


## Task 008-bugfix で実装した範囲（ChatGPT報告文の実用化）

- ChatGPT報告文セクションに「報告文をコピー」ボタンを追加
- コピー機能を実装し、1回の操作で報告文全文をクリップボードへコピー
- コピー成功時は「報告文をコピーしました」、失敗時は「コピーに失敗しました」を画面表示
- 報告文の文面を 0件 / 1件 / 2件以上 で仕様どおりに分岐
  - 0件: 記録がない案内文
  - 1件: 最新・開始情報と「前回比較不可」の案内文
  - 2件以上: 記録件数、最新・前回・開始、前回比・開始比を含む完成文面
- 体重表示を小数1桁（例: `62.3kg`）に統一
- 差分表示を符号付き表記に統一（例: `+1.0kg` / `-0.7kg` / `0.0kg`）
- 既存の「報告文作成」ボタンは、保存済みデータからの再生成専用として維持

## localStorage の保存仕様

- キー: `weightLoggerRecords`
- 値: JSON 文字列化した配列
- レコード形式:

```json
{
  "date": "2026-03-10",
  "weight": 62.3
}
```

## GitHub Pages での確認手順（最小）

1. アプリページを開く
2. 日付と体重を入力して「保存」を押す
3. 「記録一覧」に入力内容が追加されることを確認
4. ページを再読み込みし、記録が残っていることを確認
5. 「ChatGPT報告文」に件数に応じた報告文（0件/1件/2件以上）が表示されることを確認
6. 「報告文をコピー」を押し、成功メッセージが表示されることを確認

## Task 004: GitHub Pages 公開確認

このリポジトリは `index.html` がルートにある静的構成のため、GitHub Pages でそのまま公開できます。

### 公開手順（最小）

1. GitHub のリポジトリ画面で **Settings > Pages** を開く
2. **Build and deployment** の **Source** で `Deploy from a branch` を選ぶ
3. **Branch** で `main`（または公開したいブランチ）を選び、フォルダは `/ (root)` を選ぶ
4. 保存後、表示される公開 URL を開く

### 公開後の確認ポイント

- 開く URL: `https://<GitHubユーザー名>.github.io/<リポジトリ名>/`
- 成功の目安:
  - 「体重ログミニアプリ（公開確認用）」のタイトルが表示される
  - 「今日の体重入力」「記録一覧」「ChatGPT報告文」の3セクションが初期表示で見える
  - CSS が適用され、カード風の枠と余白が表示される
- 表示されない場合の確認:
  - `Settings > Pages` の Branch / Folder 設定が正しいか
  - `index.html` がリポジトリ直下にあるか
  - `index.html` の `styles.css` / `script.js` が相対パス（`styles.css`, `script.js`）のままか
  - 反映直後は数分待って再読み込みする
